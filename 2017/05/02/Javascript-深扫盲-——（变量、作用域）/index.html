Javascript 深扫盲 ——（变量、作用域）

发表于 2017-05-02 | 分类于 Study

变量

在搞变量之前，我们有必要来回忆一下JS的数据类型：

基本类型：字符串、数字、布尔、Null、Undefined；

引用类型：数组、对象

变量的创建

OK!下面我们来谈一谈变量~~~

众所周知，在JS中创建变量使用关键字：var

那么我们在创建变量的时候，JS解释器都做了什么呢？

  1. JS在解释器中创建了一个名字来保存变量；
  2. 设置变量的作用范围（作用域），这个范围就是变量创建时所在的函数；
  3. 变量提升：无论在什么位置创建变量，都会把变量提升到函数的最顶端；
  4. 赋值：在赋值时解释器会判断你所赋的值的数据类型（这就是问什么我们要回忆一下数据类型了！），
    • 如果赋的值是基本类型：那么这个值将被保存到变量中；
    • 如果赋的值是引用类型：那么这个值将会保存到内存当中，同事创建一个指向该内存的地址（引用），然后将地址保存到变量中。

变量的操作

  • 属性和方法的增加和删除
    • 基本数据类型：解释器不会报错，也不会有任何反应。
    • 引用类型：解释器会到内存中找到相应的对象，做属性和方法的增加和删除属性。
  • 变量的修改——相当于重新赋值
    • 基本数据类型：销毁原变量，重新创建变量并赋值。
    • 引用类型：如果对变量本身进行赋值操作，就相当于重新创建一个变量出来；如果对变量的属性或者方法进行修改的话，解释器会根据变量中保存的地址，找到变量的属性或者方法进行修改。

  • 变量的赋值

    • 基本数据类型：
      两个变量相互独立，没有任何连带关系。

    • 引用类型：
      由于变量中保存的是这个变量在内存中的地址，那么所复制的仅为这个变量的地址，当我们在对这个复制变量进行操作时，实际上是对该地址所对应的内存中的变量进行修改。

作用域

JS程序在运行的时候需要经历3个阶段：

  1. 词法语法的分析 —— 创建作用域，限定变量适用范围
  2. 代码生成
  3. 代码执行 —— 创建作用域链，连接了作用域。

变量作用域

一个变量的作用域是程序源代码定义这个变量的区域（作用范围）。全局变量有着全局的作用范围。然而在函数内部声明的变量，他只在函数体内有定义，也就是说他的作用范围仅为函数体内部。

变量在作用域内的特点：

  • 局部变量的优先级高于同名的全局变量。

  • 声明局部变量时必须使用var声明语句。

  • 作用域是可以嵌套的。

    我们可以把JS的全局作用域比作一个酒店，酒店里有楼层，楼层里包含房间，房间里还可以包含卫生间。

函数作用域

Javascript的函数作用域是指在函数内声明的所有变量在函数体内始终是可见的。有意思的是：这意味着变量在生命之前甚至已经可用（声明提前）。

在一些类似C语言的编程语言中，花括号内的每一点代码都具有各自的作用域，而且变量在声明它们的代码段之外是不可见的。我们称为“块级作用域”。而在Javascript（ES5）中没有块级作用域，取而代之地使用函数作用域。

ES6中的let、const命令

let命令

ES6新增了let命令，用来声明变量。它的用法类似于var。

  • 所声明的变量，只在let命令所在的代码块内（块作用域）有效。

for循环的计数器，就很合适使用let命令

  • 严格模式下（ES6强制开启严格模式），变量未声明会被报引用错误。

  • let不能在所在作用域内重复声明。

const命令

const声明一个只读的常量。

  • 一旦声明，常量的值就不能改变
  • 用const声明时，必须赋值。
  • const声明一个对象时：

这是为什么呢？明明说const声明的变量时不能修改的！你玩老子！其实不然，请脑补下在声明一个变量时，如果这个变量是引用类型，那么变量里保存的是什么呢？—— 是这个对象的引用（即在内存中的地址）。我们修改了对象，但并没有修改保存在变量中的引用啊！Understand？？！！
